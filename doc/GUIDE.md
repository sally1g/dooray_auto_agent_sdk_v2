## 1. 전체 흐름 한눈에 보기

```
① Dooray에 수정 요청 게시글 작성 (상태: 할일)
        ↓ 자동 감지
② Claude가 작업 시작 (상태: 처리중, 2~4분 소요)
        ↓
    ┌─────────────────────────────────┐
    │ 성공                       실패 │
    ↓                                ↓
③ GitHub에 push 완료          ③ 처리실패 상태로 변경
   (상태: 구현)                    + 오류 댓글 안내
        ↓
④ 테스터 검수
    ├─ 통과 → 운영배포대기 상태로 변경 (수동)
    └─ 재작업 필요 → 댓글 작성 후 보완요청/할일로 변경
                        ↓ 자동 재트리거
                    ② Claude 재작업
        ↓ (운영 반영 후)
⑤ 완료 상태로 변경 → 작업 브랜치 자동 삭제
```

## 2. Dooray 게시글 작성 방법

### 프로젝트
- **Dooray 프로젝트:** `AI이슈관리-칸반-테스트중`
- **게시글 등록 시 상태:** 반드시 `할일`로 설정

### 게시글 본문 양식 (비개발자용)

```
## 어느 화면인가요?
- URL: https://www.example.com/jobs
- 메뉴 경로: 채용정보 > 목록

## 화면 어디에 있나요?
페이지 상단 배너 텍스트

## 지금 어떻게 보이나요?
채용공고 목록

## 어떻게 바뀌어야 하나요?
최신 채용공고

## 왜 바꾸나요?
운영팀 요청 — 표현 통일
```

> **팁:** URL, 화면에 보이는 텍스트를 정확히 적을수록 Claude가 올바른 파일을 찾습니다.  
> 파일 경로나 코드를 몰라도 됩니다.

## 3. 상태(칸반) 의미와 전이 규칙

| 상태 | 의미 | 전이 주체 |
|------|------|-----------|
| **할일** | 작업 대기 중 | 게시글 최초 등록 / 수동 복귀 |
| **처리중** | Claude가 작업 중 (건드리지 마세요) | 자동 (봇) |
| **구현** | GitHub push 완료, 테스터 검수 필요 | 자동 (봇) |
| **처리실패** | 자동 처리 실패, 수동 개입 필요 | 자동 (봇) |
| **보완요청** | 테스터가 추가 수정 요청 | 수동 (테스터) |
| **운영배포대기** | 검수 완료, 운영 반영 대기 | 수동 (테스터) |
| **완료** | 운영 반영 완료 → 브랜치 자동 삭제 | 수동 (담당자) |

> ⚠️ **처리중 상태일 때는 게시글을 수정하거나 상태를 바꾸지 마세요.**  
> Claude가 작업 중이며 2~4분 내로 완료됩니다.

## 4. 트리거 조건 — 언제 자동 실행되나?

### 최초 실행
| 조건 | 동작 |
|------|------|
| 게시글 등록 + 상태 `할일` | Claude 자동 실행 |

### 재트리거 (두 조건 모두 충족해야 함)
| 조건 1 (상태) | 조건 2 (댓글) | 동작 |
|--------------|--------------|------|
| `보완요청` 또는 `할일` | 사람이 쓴 댓글이 있음 | Claude 재실행 |

**재트리거 시나리오 예시:**

```
시나리오 A: 상태 변경 먼저
  댓글 작성 → 보완요청 상태로 변경 → 자동 재트리거 ✅

시나리오 B: 댓글 먼저
  보완요청 상태로 변경 → 댓글 작성 → 자동 재트리거 ✅
```

### 자동 무시되는 이벤트
- `처리중` / `구현` / `처리실패` / `운영배포대기` 상태의 댓글
- 댓글 없이 상태만 변경
- 봇이 작성한 결과 댓글

## 5. 브랜치 정책

### 기준 브랜치
```
TARGET_BETA_BRANCH = dev_kwonay   (.env 설정값)
```

모든 작업은 **`dev_kwonay` 브랜치를 기준**으로 시작합니다. `main`은 절대 직접 건드리지 않습니다.

### 작업 브랜치 네이밍
```
auto/dooray-{task_id}
예) auto/dooray-4345865361621016640
```

### 브랜치 생성~삭제 전체 흐름

```
1. dev_kwonay 최신화
   git fetch origin dev_kwonay

2. 작업 브랜치 생성 (또는 재사용)
   git checkout -b auto/dooray-{task_id}
   (재작업 시 기존 브랜치 이어서 사용)

3. 코드 수정 + 검증
   npx tsc --noEmit
   npm run lint
   npm test

4. 커밋 & rebase
   git rebase origin/dev_kwonay

5. GitHub push
   git push origin auto/dooray-{task_id}

6. dev_kwonay에 병합
   git checkout dev_kwonay
   git merge --no-ff auto/dooray-{task_id}
   git push origin dev_kwonay

7. 완료 상태 전이 시 브랜치 자동 삭제
   git push origin --delete auto/dooray-{task_id}
```

### GitHub에서 확인
- **작업 브랜치:** `auto/dooray-{task_id}` (구현 상태)
- **반영 브랜치:** `dev_kwonay` (구현 완료 후)
- **운영 브랜치:** `main` (수동 운영 배포, 봇 미관여)

## 6. 결과 확인 방법

### Dooray 댓글 (자동 작성)
작업 완료 후 게시글에 자동으로 결과 댓글이 달립니다.

**성공 예시:**
```
## ✅ 자동 처리 완료 → 상태: 구현

이벤트: task.created
태스크 ID: 4345865361621016640
테스트 결과: pass
커밋: abc1234
브랜치: auto/dooray-4345865361621016640 → dev_kwonay

### 수정 파일
- app/jobs/page.tsx

### 요약
메인 페이지 타이틀을 'Hello World'에서 'Hello Dooray!'로 변경했습니다.
```

**실패 예시:**
```
## ❌ 자동 처리 실패 → 상태: 처리실패

### 오류
TypeScript 타입 오류로 빌드 실패. 수동 검토 필요.
```

### GitHub
- `incruit-git/FairWorks` 저장소
- `dev_kwonay` 브랜치에서 커밋 확인

## 7. 재작업 요청 방법

### 방법 1: 상태 변경 후 댓글
1. 칸반에서 상태를 `보완요청`로 변경
2. 게시글에 추가 지시 댓글 작성

### 방법 2: 댓글 후 상태 변경
1. 게시글에 추가 지시 댓글 작성
2. 칸반에서 상태를 `보완요청`(또는 `할일`)로 변경

### 재작업 댓글 작성 요령
```
변경 전: "버튼 색을 파란색으로 바꿔주세요"  ← 너무 간단
변경 후:
  - 어느 버튼: 채용공고 목록 페이지 상단 '검색' 버튼
  - 현재: 회색(#999999)
  - 변경 요청: 파란색(#0066CC)
  - 참고: 디자인 가이드의 primary 색상 사용
```

> Claude는 최신 댓글을 기존 요청보다 우선하여 반영합니다.

## 8. 자주 묻는 질문

**Q. 처리중 상태가 오래 유지됩니다.**  
A. 정상입니다. Claude가 파일 탐색 → 수정 → 타입체크/린트 → GitHub push 순서로 작업하며 **2~4분** 소요됩니다. 기다려 주세요.

**Q. 처리실패 상태가 됐습니다.**  
A. 댓글에 오류 내용이 적혀 있습니다. 내용 확인 후 요청 본문을 보완하고 댓글로 추가 지시를 남긴 뒤 `할일` 상태로 변경하면 재시도합니다.

**Q. 같은 화면 수정 요청이 동시에 여러 개 들어오면?**  
A. 한 번에 하나씩 순서대로 처리됩니다. 앞 작업이 완료된 후 다음 작업이 시작됩니다.

**Q. 운영(main)에는 언제 반영되나요?**  
A. 봇은 `dev_kwonay`까지만 자동 반영합니다. `dev_kwonay` → `main` 운영 배포는 담당자가 수동으로 진행합니다. 테스터가 검수 완료 후 `운영배포대기` 상태로 변경하면 개발자에게 배포 신호가 됩니다.

**Q. 특정 파일은 건드리지 말아야 하는데 어떻게 하나요?**  
A. 게시글 본문 또는 댓글에 "○○ 파일은 수정하지 말 것"을 명시하면 Claude가 해당 지시를 따릅니다.
