# Dooray 칸반보드 사용 가이드 (일반 사용자용)

> Dooray에 수정 요청을 작성하면 AI가 자동으로 코드를 수정해 드립니다.

---

## � 시작하기 전에

- **이슈 관리:** 프로젝트 상단의 **'보드'** 뷰를 사용하면 모든 이슈를 한눈에 볼 수 있습니다.
- **이슈 등록:** 각 이슈는 **'게시글'**로 관리됩니다.
- **추가 요청:** 해당 이슈에 대한 추가 요청사항은 **'댓글'**로 작성합니다.

> 💡 보드 뷰를 사용하면 이슈 상태를 드래그앤드롭으로 쉽게 변경할 수 있습니다.

---

## �📝 작업 요청 방법

### 1. Dooray 게시글 작성
- **프로젝트:** `AI이슈관리-칸반-테스트중`
- **상태:** `할일`로 설정
- **본문 양식:**

```
## 어느 화면인가요?
예) https://www.example.com/jobs (채용정보 목록)

## 화면 어디에 있나요?
예) 페이지 상단 배너 텍스트

## 지금 어떻게 보이나요?
예) "채용공고 목록"

## 어떻게 바뀌어야 하나요?
예) "최신 채용공고"

## 왜 바꾸나요?
예) 운영팀 요청 - 표현 통일
```

> **💡 팁:** URL과 화면에 보이는 텍스트를 정확히 적으면 AI가 더 잘 찾습니다.

---

## 🔄 상태(칸반) 이해하기

| 상태 | 의미 | 누가 변경하나요? |
|------|------|-----------------|
| **할일** | 작업 대기 중 | 사용자 (게시글 등록) |
| **처리중** | AI가 작업 중 (2~4분 소요) | 자동 ⚡ |
| **구현** | 코드 수정 완료, 테스트 필요 | 자동 ⚡ |
| **처리실패** | 오류 발생, 요청 수정 필요 | 자동 ⚡ |
| **보완요청** | 추가 수정 필요 | 테스터 |
| **운영배포대기** | 테스트 완료, 운영 반영 대기 | 테스터 |
| **완료** | 운영 반영 완료 | 담당자 |

> ⚠️ **주의:** `처리중` 상태일 때는 게시글을 수정하거나 상태를 변경하지 마세요.

---

## ⚙️ 새 업무 등록 시 필수 설정

게시글 작성 외에 다음 항목들을 반드시 설정해야 합니다:

### 1. 템플릿 선택
- **선택 예시:** `기능수정`

### 2. 담당자 지정
- 작업 담당자를 지정해 주세요

### 3. 대상 서비스 선택
업무가 어느 서비스에 해당하는지 선택합니다:

| 서비스 | 설명 | 사용 빈도 |
|--------|------|----------|
| **tenant-app** | 일반 박람회 홈페이지, 박람회 ADMIN | ⭐ 대부분의 수정사항 |
| **platform-admin** | FairWorks Admin | 관리자 기능 수정 |

> 💡 **대부분의 경우 `tenant-app`을 선택**하시면 됩니다.

## 🔁 재작업 요청하기

### 방법
1. **댓글 작성:** 추가로 수정할 내용을 댓글에 작성
2. **상태 변경:** `보완요청` 또는 `할일`로 변경

**좋은 재작업 요청 예시:**
```
- 어느 버튼: 채용공고 목록 페이지 상단 '검색' 버튼
- 현재: 회색(#999999)
- 변경 요청: 파란색(#0066CC)
```

**나쁜 예시:**
```
버튼 색 바꿔주세요  ← 어느 버튼인지 불명확
```

## ✅ 결과 확인하기

작업 완료 후 게시글에 자동으로 댓글이 달립니다.

**성공 예시:**
```
## ✅ 자동 처리 완료 → 상태: 구현

### 수정 파일
- app/jobs/page.tsx

### 요약
메인 페이지 타이틀을 'Hello World'에서 'Hello Dooray!'로 변경했습니다.
```

**실패 예시:**
```
## ❌ 자동 처리 실패 → 상태: 처리실패

### 오류
파일을 찾을 수 없습니다. URL을 다시 확인해 주세요.
```

## ❓ 자주 묻는 질문

**Q. 처리중 상태가 계속 유지됩니다.**  
A. 정상입니다. 2~4분 정도 기다려 주세요.

**Q. 처리실패 상태가 됐어요.**  
A. 댓글에 오류 내용을 확인하고, 게시글 내용을 보완한 뒤 댓글로 추가 설명을 달아주세요. 그 다음 상태를 `할일`로 변경하면 재시도됩니다.

**Q. 여러 개를 동시에 요청하면?**  
A. 한 번에 하나씩 순서대로 처리됩니다.

**Q. 실제 운영 사이트에는 언제 반영되나요?**  
A. 테스터가 검수 완료 후 `운영배포대기` 상태로 변경하면, 담당자가 수동으로 운영 반영합니다.
